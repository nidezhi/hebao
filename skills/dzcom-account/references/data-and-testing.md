# 账户域数据规则与验收

## 1. 权威结构

账户域结构唯一来源：

- `src/main/resources/db/migration/V1__account_domain.sql`
- `docs_new/04-initial-database-design.md`

账户域包含 `aiw_user`、`aiw_user_identity`、`aiw_user_credential`、`aiw_user_profile`、`aiw_user_risk_profile`、`aiw_user_role`、`aiw_user_preference`。

不得恢复旧版把用户名、邮箱、手机号、密码和风险等级全部放在 `aiw_user` 的设计。

## 2. 数据标准化

- `USERNAME`：去除首尾空格，首版按数据库排序规则不区分大小写。
- `EMAIL`：去除首尾空格并转小写。
- `PHONE`：持久化前标准化为 E.164。
- 空字符串转为 `null`，必填登录标识不得为空。
- 登录和唯一性判断只使用 `normalized_value`。
- `identity_value` 仅用于必要展示，接口输出按权限脱敏。
- 密码只保存到 `secret_hash`，哈希算法及版本必须显式记录。

## 3. 唯一性与生命周期

- 用户编号由 `uk_aiw_user_user_no` 保证唯一。
- 登录标识由 `identity_type + normalized_value` 保证全局唯一。
- 每个用户每种登录标识、凭据类型、资料和风险画像只有一条当前记录。
- 同一用户、角色、作用域只有一条分配记录。
- 同一用户同一偏好键只有一条记录；删除后再次设置应恢复原记录。
- 软删除不释放登录标识，禁止新用户复用历史身份。

## 4. 无外键一致性

- 所有 `user_biz_id` 都是逻辑引用，不存在数据库外键。
- 注册和删除在账户域本地事务中显式操作多张表。
- 仓储不得依赖对象级联保存或删除。
- 应提供一致性检查，发现无用户主体的标识、凭据、资料、角色和偏好。
- 跨领域清理不在账户删除事务中直接修改其他领域表，应发布领域事件并由对应领域处理。

## 5. 事务边界

- 注册：多表保存属于同一事务，唯一索引处理并发竞争。
- 登录：失败次数、锁定状态、成功时间在数据库事务内更新；会话在提交后创建。
- 禁用和锁定：用户状态提交后撤销会话，失败必须重试或告警。
- 改密：更新哈希并递增凭据版本，提交后撤销旧会话。
- 删除：账户域全部表显式软删除，提交后撤销会话并发布 `AccountDeleted` 事件。
- 偏好 upsert：查询和更新在一个事务中完成，数据库唯一约束兜底。

## 6. 单元测试

至少覆盖：

- 注册默认状态、KYC、风险等级和角色。
- 登录标识标准化。
- 非法状态变化和非法风险等级。
- 禁用、锁定和删除用户不能登录。
- 密码失败次数、锁定时间和成功后清零。
- 凭据版本在改密后递增。
- 状态重复变更的幂等行为。

## 7. 仓储集成测试

至少覆盖：

- 按用户名、邮箱、手机号的类型和标准化值查询。
- 常规查询排除软删除记录。
- 用户详情多表组装。
- 用户列表分页、筛选、排序和敏感字段脱敏。
- 登录标识并发唯一约束冲突。
- 多仓储注册事务失败后整体回滚。
- 偏好和角色的 upsert、恢复与软删除。
- 不使用外键时的一致性巡检查询。

## 8. API 集成测试

至少覆盖：

- 注册成功和重复标识注册。
- 登录成功、密码错误、凭据锁定、禁用用户登录。
- Cookie 会话创建、解析和登出。
- 未登录访问受保护接口。
- 本人资料更新不能修改状态、角色、KYC 和风险等级。
- 管理端列表、启用、禁用、锁定、角色分配和删除。
- 禁用、改密和删除后的旧会话失效。

## 9. 推荐实现顺序

1. 账户枚举、值对象和标识标准化。
2. 七张表的 Entity、仓储和显式映射。
3. User、LoginIdentity、UserCredential 领域规则。
4. 注册用例及多表事务。
5. Redis 会话和登录、登出、当前用户。
6. 本人资料、登录标识和密码。
7. 管理端列表、详情、状态、角色、KYC、风险等级和删除。
8. 用户偏好、审计、领域事件和一致性巡检。
9. OpenAPI 与完整测试。

## 10. 本轮非目标

- 不实现真实 KYC 审核。
- 不实现短信、邮件验证码。
- 不实现 OAuth 协议流程。
- 不实现权限定义和复杂 RBAC 管理后台。
- 不实现找回密码。
