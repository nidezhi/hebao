# 账户域业务边界

## 1. 领域职责

账户域负责：

- 用户注册、登录、登出与会话管理。
- 用户主体生命周期和登录标识管理。
- 密码凭据、失败次数、临时锁定和凭据版本。
- 用户展示资料、KYC 状态、风险等级、角色和偏好。
- 管理端用户查询、启用、禁用、锁定和软删除。

账户域暂不负责：

- 短信、邮件发送和第三方 OAuth 协议接入。
- 身份证、人脸等完整 KYC 审核流程。
- 角色权限配置后台和组织体系。
- 找回密码与验证码流程。
- 用户资金账户和投资组合。

## 2. 统一语言

| 术语 | 含义 |
| --- | --- |
| 用户主体 | 平台身份和生命周期载体，对应 `aiw_user` |
| 用户编号 | 面向运营展示且创建后不变的 `user_no` |
| 登录标识 | 用户名、邮箱、手机号等可认证标识 |
| 凭据 | 密码哈希及其算法、版本和锁定信息 |
| 业务 ID | 跨表逻辑关联使用的 UUID `biz_id` |
| 风险画像 | KYC 状态、风险等级及测评版本 |
| 会话 | 登录成功后保存在 Redis 的可撤销状态 |

## 3. 领域对象

### User 聚合根

只包含稳定生命周期信息：

- `bizId`
- `userNo`
- `status`
- `version`
- `registeredAt`
- `lastLoginAt`
- `createdAt`
- `updatedAt`
- `deleted`
- `deletedAt`

核心行为：

- `register`
- `recordSuccessfulLogin`
- `enable`
- `disable`
- `lock`
- `delete`
- `canLogin`

不得通过 Lombok `@Data` 暴露任意 setter。状态变化必须经过领域行为。

### LoginIdentity

表示一个用户的登录标识，包含标识类型、原始值、标准化值、验证状态和可用状态。首版支持 `USERNAME`、`EMAIL`、`PHONE`，未来可增加 `OAUTH_*` 类型。

### UserCredential

隔离认证秘密，包含凭据类型、哈希、算法、凭据版本、失败次数和锁定截止时间。任何领域事件、日志和 API 响应都不得包含 `secretHash`。

### UserProfile

保存昵称、头像、语言和时区。邮箱、手机号属于登录标识，不放入资料表。

### UserRiskProfile

保存 KYC 状态、风险等级、测评版本和审核时间。风险等级只能由风险测评或管理用例修改。

### RoleAssignment 与 UserPreference

角色分配和偏好是账户域独立实体。偏好键必须受白名单或命名规范约束；角色只保存分配结果，不承载权限定义。

## 4. 聚合协调原则

- 领域模型可以表达完整账户概念，但持久化不得使用一个 JPA 聚合对象图。
- 注册用例在一个本地事务内显式保存用户主体、登录标识、密码凭据、默认资料、风险画像和默认角色。
- 查询详情由查询服务组装多表数据，不能让 Controller 分别调用多个仓储。
- 删除用例显式软删除账户域各表；不存在数据库外键和级联删除。
- 会话在数据库事务提交后撤销，失败时记录可重试任务或告警。

## 5. 状态规则

### 账户状态

| 代码 | 枚举 | 规则 |
| --- | --- | --- |
| `0` | `DISABLED` | 不能登录，现有会话必须撤销 |
| `1` | `ACTIVE` | 满足凭据规则时可以登录 |
| `2` | `LOCKED` | 不能登录，可由安全或管理用例解锁 |

### KYC 状态

| 代码 | 枚举 |
| --- | --- |
| `0` | `UNVERIFIED` |
| `1` | `VERIFIED` |
| `2` | `REVIEWING` |
| `3` | `REJECTED` |

### 风险等级

- 有效值为 `1..5`。
- 用户不能通过个人资料接口直接修改。
- 由风险测评或管理端专用用例修改。

## 6. 权限边界

公开访问：注册、登录。

登录用户：登出、查询本人、更新本人资料、维护本人登录标识、修改密码、维护本人偏好。

管理角色：用户列表与详情、创建用户、启用、禁用、锁定、软删除、角色分配、修改 KYC 状态和风险等级。

权限能力未实现时不得临时开放管理接口，必须先提供 `CurrentOperator` 和权限校验契约。

## 7. 删除规则

- 删除采用软删除，不物理删除账户域记录。
- 登录标识、凭据、资料、风险画像、角色和偏好由应用服务显式标记删除。
- 删除后立即撤销全部会话。
- 删除不释放历史用户名、邮箱和手机号，避免审计身份被重新占用。
- 删除操作写入 `aiw_audit_log`，审计详情必须脱敏。
