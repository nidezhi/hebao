# 24 Frontend UI Aliyun Console Reference

生成日期：2026-07-01

本文是 dzcom_web 前端 UI 方向基线。2026-07-01 已确认采用阿里云控制台式方向，后续业务页面按本文逐步重构。

## 最新结论区

### 定位

- UI 方向参考阿里云控制台类产品：清爽、克制、宽广、信息密度高、面向长期运维和反复操作。
- 当前项目是投资闭环运营工具，不做营销页、炫技页或大面积装饰性视觉。
- 全局框架先统一为全宽薄顶栏、约 `204px` 左侧菜单和宽工作面；业务页面再逐页改造并执行 type-check、build、目标路由冒烟。
- 本轮没有直接写入 Figma 文件；产物是可导入 Figma 的 SVG 框架图和前端 `/ui-system` 预览页。

### 布局骨架

- 顶部栏：全宽薄顶栏，产品名、全局搜索、账号和全局入口；高度建议 `50px` 左右。
- 左侧导航：从顶栏下方开始，按业务域分组，父级约 `40px` 高、子级约 `34px` 高且明显内缩；当前项使用浅灰底和黑色文字；宽度建议约 `204px`，折叠宽度约 `50px`。
- 内容标题区：页面标题、面包屑、对象上下文和主操作；避免巨大 hero。
- 工具条：高频筛选一行平铺，按钮靠右，低频筛选收进“更多”；不得挤占主要工作区。
- 主工作区：标签页 + 表格 + 大留白为主，结构化详情通过抽屉或局部详情承载。
- 右侧检查器：只在确实需要对象追踪时出现，不作为所有页面默认结构。

### 组件规则

- 卡片圆角不超过 `8px`，列表页优先不用卡片包住整块内容，避免“卡片墙”。
- 列表、按钮、输入框、弹窗统一采用控制台比例：按钮/输入约 `32px` 高，表格行约 `44px`，弹窗头/底部有清晰分割线。
- 筛选区默认一行紧凑排列，超过宽度自然折行；高级筛选默认收起，按钮靠右。
- 表格优先承载主要数据，表头浅灰底，空数据允许大面积留白；原始编码只放小字、Tooltip 或详情追踪字段。
- 配置页采用左侧树/分组 + 右侧表单/列表/预览，不把所有配置平铺在一个长页里。
- 运行态页面采用“状态条 + 任务流 + 当前对象详情 + 审计记录”结构，刷新频率和时间戳必须可见。

### 禁止项

- 禁止大面积深色科幻背景、霓虹网格、玻璃拟态作为默认业务页面风格。
- 禁止营销式 hero、夸张渐变、装饰性大卡片堆叠。
- 禁止使用阿里云品牌色、紫色高饱和主题或阿里云图标作为 DZAI 品牌符号；站点 icon 必须使用 DZAI 自有黑灰标识。
- 禁止主展示区裸露后端英文枚举，例如 `INVESTMENT_REPORT`、`DEFAULT`。
- 禁止为了视觉效果降低信息密度、隐藏错误原因或减少关键操作反馈。
