# 05 Cross Repo Change Checklist

生成日期：2026-06-28

本文是每个前后端联动功能的执行清单。

## 最新结论区

### 标准推进顺序

1. **先读取并声明本次适用的后端/前端开发铁律；铁律是第一优先级，不满足不得进入代码实现。**
2. 先更新共享 handoff/API contract。
3. 后端补接口、DTO、数据库迁移和测试。
4. 前端补 API client、类型、adapter、字典和页面体验。
5. 如果接口不满足前端铁律，写入 `06-backend-gap-list.md`。
6. 收尾时同时跑后端验证、前端验证、至少一个真实链路冒烟。
7. 完成后把结果写回 `00-current-handoff.md`，后续对话默认不再读取已完成过程上下文。

| 问题 | 结果 |

| 最小真实链路冒烟是什么？ | 待填写 |

### 后端完成检查

- 是否逐条满足 `03-backend-laws.md` 相关铁律；未满足项是否已写入 gap/contract？
- Controller 路径、方法、请求体、响应体明确。
- Request 参数校验完整。
- DTO 包含前端展示所需 display/summary/status/error context。
- Service 状态机、幂等、权限、风控边界明确。
- DB migration 可重复执行基线验证，字段默认值和索引明确。
- 测试覆盖主要成功、失败、权限或状态冲突场景。

### 前端完成检查

- 是否逐条满足 `02-frontend-laws.md` 相关铁律；未满足项是否已写入后端 gap/contract？
- API client 路径集中维护。
- model 与后端 DTO 对齐。
- adapter 将后端结构转为页面稳定展示模型。
- 对象 id 输入使用选择器或可搜索下拉。
- 页面核心模块结构化展示，不以 JSON 为主。
- type-check、build、目标路由冒烟通过。
