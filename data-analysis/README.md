# 数据分析资料目录

生成日期：2026-07-01

## 目录边界

本目录只保存数据质量验证、核心闭环支撑能力、Skill、Prompt、节点、模型和架构更新建议。

本目录内容是参考意见，不代表已实施的业务配置，不会直接改变后端运行行为。

本轮严格不修改：

- `src/main/java`
- `src/test/java`
- `src/main/resources/db/migration`
- `config`
- 前端仓库

## 核心目标

围绕 `docs_new/00-current-handoff.md` 中的当前闭环状态，使数据能够支撑核心功能实现：

```text
可信数据源
  -> 采集与数据治理
  -> 投资报告
  -> Prompt / Skill / 模型治理
  -> Mock 组合与模拟交易
  -> 回测、反馈、风控审计
  -> 驾驶舱展示
  -> 数据源、Prompt、模型和 Skill 持续优化
```

数据质量不足时，只能输出数据缺口、风险提示和下一步治理动作，不能伪装成确定投资建议。

## 当前判断

现有系统已经具备基础闭环、数据质量快照、模型调用审计、持续进化分析和一批 AI Skill 资产。下一阶段不应简单扩大模型调用，而应新增或强化“数据质量诊断、缺口归因、采集器字段映射审查、报告就绪门禁、闭环证据核对、成本失败熔断”这些治理型 Skill。

这些 Skill 的运行边界必须清晰：

- 可以发现数据质量问题。
- 可以提出数据、架构、Prompt、Skill、节点、模型绑定更新方案。
- 可以生成候选配置和人工审核包。
- 不直接写核心产品、行情、资讯、报告、交易或风控业务表。
- 不绕过质量门禁、人工确认和灰度开关。
